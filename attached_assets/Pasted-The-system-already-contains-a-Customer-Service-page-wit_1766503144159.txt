The system already contains a Customer Service page with an existing
“Add Item” field that selects products from Inventory.

Update the behavior of this existing “Add Item” field to support
roll-based inventory deduction.

Inventory Context:
Products available in Inventory:
- ELITE
- GARWARE MATT
- GARWARE PLUS
- GARWARE PREMIUM

Each product has multiple rolls.
Each roll has:
- Roll ID
- Product
- Unit (Meters or Square KM)
- Available Quantity
- Status (Available / Finished)

------------------------------------------------
Add Item (Inventory) – Updated Behavior
------------------------------------------------

1. Product Selection (Existing Behavior):
- User selects a Product using the existing Add Item field
- Product list remains unchanged

2. Roll Selection (New Behavior):
- After Product is selected, system must require selecting a Roll
- Roll list must be filtered to show ONLY:
  - Rolls belonging to the selected Product
  - Rolls with Available Quantity > 0
  - Rolls with Status = Available

Roll dropdown must display:
- Roll ID
- Available Quantity
- Unit

Example display:
"GP-R3 | 180 meters available"

3. Auto-Filled Fields:
- Unit is automatically filled from the selected Roll
- Unit field is read-only
- Available Quantity is shown for reference only

4. Quantity Used:
- User enters the quantity to be consumed from the selected roll

------------------------------------------------
Validations
------------------------------------------------

- Quantity Used must be <= Roll.Available Quantity
- Selected Roll must belong to the selected Product
- Unit must not be editable
- If validation fails, Add Item cannot be saved

------------------------------------------------
Inventory Update Logic
------------------------------------------------

When the Customer Service record is saved or confirmed:

For each Add Item entry:
- Deduct Quantity Used from the selected Roll’s Available Quantity

Formula:
Roll.Available Quantity = Roll.Available Quantity - Quantity Used

If Available Quantity becomes 0:
- Update Roll Status to Finished
- Exclude roll from future Add Item selections

Inventory updates must be atomic:
- If any Add Item fails validation, no roll quantities are updated

------------------------------------------------
Usage Tracking
------------------------------------------------

For every Add Item entry, store:
- Product
- Roll ID
- Unit
- Quantity Used
- Customer Service reference
